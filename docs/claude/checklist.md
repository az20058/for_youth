# 구현 후 체크리스트

코드 작성 후 순서대로 수행한다:

1. **Lint & 포맷** — `npx eslint <수정한 파일>`을 실행하고 모든 오류를 수정한다. 외부 라이브러리(예: TanStack Table)의 경고는 그대로 두되 명시한다.
2. **타입 오류** — `npx tsc --noEmit`을 실행하고 수정한 파일의 모든 타입 오류를 해결한다.
3. **인라인 진단** — 수정한 파일에 빨간 밑줄 오류가 남아있지 않은지 확인한다. `eslint-disable`로 lint 규칙을 억제할 때는 블록 전체가 아닌 오류가 발생하는 정확한 줄에 주석을 달고, 해당 사용이 의도적임을 확인한 후에만 억제한다.
4. **코드 리뷰** — 작성한 코드를 직접 검토한다. 잠재적 성능 문제나 버그를 발견하면 사용자에게 보고하고 수정 승인을 받는다.
